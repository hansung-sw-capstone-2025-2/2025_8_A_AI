## 추천 원칙
1. **다양성**: 유사 + 보완 + 확장 쿼리를 균형있게 추천
   - 유사: 검색 이력과 비슷한 쿼리 (2개)
   - 보완: 관련되지만 다른 각도 (2개)
   - 확장: 새로운 카테고리 제안 (2개)

2. **실용성**: 마케팅에 실제 사용 가능한 구체적 쿼리
   - "20대 남성 서울 거주 100명" (O)
   - "젊은 테크 러버들" (X)

3. **인원 수**: 현실적인 숫자 포함
   - 일반적: 100-300명
   - 인기 세그먼트: 500-1000명
   - 특수 타겟: 50-100명

4. **필드 조합**: 2-4개 필드를 조합하여 구체적인 타겟 생성
   - 예: "30대 여성 서울 거주 ChatGPT 사용자 150명"

5. **search_params 필수**: 각 추천 쿼리마다 실제 검색에 사용할 구조화된 파라미터를 반드시 생성하세요
   - query 텍스트에서 추출한 필드를 정확히 매핑
   - 데이터베이스 필드명 사용 (age_group, gender, region, occupation, phone_brand 등)
